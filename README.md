# ai-hackathon-ppc
Our team's work on the special challenge by ΔΕΗ/PPC: Vision-to-RAG Prompt Builder for Energy Billing. 

# ΑΡΧΙΚΟΣ ΟΔΗΓΟΣ 

1) Clone the repo
2) Φτιαχτε ενα python virtual environment με: python -m venv venv
3) Δειτε στο pyvenv.cfg μεσα στο venv folder οτι εχετε Python 3.12.6 γιατι χρησιμοποιουμε πολλες βιβλιοθηκες που μπορει να σπανε απο εκδοση σε εκδοση.
4) Το script για να μπαινουμε στο environment ειναι: .\venv\Scripts\activate   η κατι τετοιο τελοσπαντων
5) Καντε pip install -r requirements.txt για να κατεβουν οι βιβλιοθηκες στο virtual environment. Καντε το με καινουρια εκδοση του Pip για να μην εχουμε θεμα και με αυτο
6) Επισησς φτιαχτε ενα .env αρχειο στο οποιο μεσα βαλτε GOOGLE_API_KEY=AIzaS...   οπου θα χρειαστει να δημιουργησετε ενα free tier api key για gemini. Εντελως δωρεαν και δεν σε χρεωνει αν ξεπερασεις οριο απλα σταματαει. Νομιζω google ai studio -> get api key -> free tier κατι τετοιο.
7) Τρεχτε python ingest.py το οποιο ουσιαστικα φτιαχνει το Knowledge_base στο οπιιο θα βασιστει το μοντελο για την απαντηση με RAG.
8) Αν ολα πανε καλα αν τρεξετε streamlit run app.py --server.port 8888 θα αρχισει μπορειτε να δειτε το ντεμο.
9) Εχω φορτωσει ελαχιστα mock data οποτε αν παρετε ενα screenshot το παρακατω και το δωσετε εκει που λεει να ανεβασετε μια φωτογραφια θεωρητικα θα δουλεψει.

# ΠΕΡΙΓΡΑΦΗ

Τον κωδικα γενικα τονεχω κανει σχεδον εξ ολοκληρου με ai. Ηθελα απλα να δειτε πως πειρπου θα ειναι το προτζεκτ αν χρησιμοποιησουμε langchain + streamlit.

1) Στον φακελο data/ θα εχουμε το αρχειο με το customer database/warehouse
2) Στον φακελο knowledge_base/ θα εχουμε καποια txt / pdf κλπ. απο τα οποια το script ingest.py που βρισκεται στο root του προτζεκτ θα δημιουργει το vector database στο chroma_db αρχειο μεσα στο knowledge_base/
3) Το κυριο frontend ειναι στο app.py και χρησιμοποιει συναρτησεις απο τον φακελο utils/helpers.py οποτε οτι δουλεια θα συμβαινει για frontend θεωρητικα εκει θα γινεται.
4) Η κυρια λογικη του προτζεκτ ειναι στο main.py το οποιο χρησιμοποιει τα scripts στον φακελο src/. Αυτα ειναι κατα βαση οτι μας ζητανε σπασμενο σε κομματια.
5) Εξηγω συντομα τι κανει το καθενα σε σχολια
6) Το ΑΙ εβαλε στο φροντεντ ενα demo control και καλα λογικη για να ελεγχουμε οταν κανουμε το demo καποια πραγματα δεν ξερω αν μας χρειαζεται οποιος παρει τελικα το φροντεντ το βλεπει αυτο.
